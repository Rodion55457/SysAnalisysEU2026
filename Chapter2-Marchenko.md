# Cистемний аналіз та проектування інформаційних систем.
## Марченко Родіон 241/з

**Основні вимоги:**

- Система для керування замовленнями створюється для застосування у закладах громадського харчування для автоматизації та оптимізації процесу прийому заявок, передачі іноформації про бажані страви на кухню та відслідковування статусу виконання кожного замовлення.

- Система має мати чотири основних користувацьких інтерфейси:
  - Інтерфейс для касира для перегляду поточного доступного меню, вибору страв за бажанням клієнта й формування замовлення.
  - Інтерфейс для клієнта з можливістю перегляду поточного меню на екрані.
  - Інтерфейс для кухаря, що показуватиме інформацію про замовлення, що очікують на приготування з можливістю реєстрації статусу замовлення (очікує черги, готується, готово).
  - Інтерфейс для менеджера з можливістю перегляду звіту про надані послуги з розбиттям по замовленням та з загальними фінансовими показниками наданих послуг по всім замовленням або обраним категоріям, а також можливістю експорту фінансових даних у формат, сумісний з ПЗ для бухгалтерського обліку.

- Система має мати можливість працювати на терміналах з різними способами вводу, такими як клавіатура (для касира та менеджера) та сенсорний екран (для кухаря та для відвідувача).

- Кожен тип інтерфейсу має мати можливість базової кастомізації, такої як зміна шрифта, підлаштування під розмір та орієнтацію екрану та позиції віртуальних кнопок у випадку сенсорних інтерфейсів.

- В системі має бути наявна можливість розподілення прав доступу до її компонентів та доступних для перегляду даних по ролям: клієнт, касир, кухар, менеджер, адміністратор. Режим адміністратора для налагодження та діагностики системи та режим менеджера з доступом до ділових таємниць мають бути захищені автентифікацією з паролем. Клієнт не має мати можливості збільшення своїх привілеїв будь яким чином для уникнення атак на систему ззовні: статус теміналу як клієнтського має визначатися на етапі розгортання й налаштування системи.

- Структура системи: стандартна клієнт-серверна архітектура: сервер відповідає за програмну логіку (back end) та бази даних, технічно є звичайним ПК на архітектурі х86 й має бути встановлений в адміністративній частині закладу; клієнти відповідають за відображення інтерфейсів та взаємодію з користувачами, фізично є тонкими клієнтами невеликої потужності. Дані між клієнтом та сервером передаються по локальній мережі Ethernet.
